# ArtePay PWA Setup

Your ArtePay application is now a fully functional Progressive Web App (PWA)!

## What's Included

### 1. PWA Manifest (`public/manifest.json`)
- App name: **ArtePay**
- Theme color: `#10B981` (Green)
- Display mode: Standalone
- Icons in multiple sizes (192x192, 512x512, SVG)
- App shortcuts for quick access to Boards

### 2. Service Worker
- Automatically generated by next-pwa
- Provides offline functionality
- Caches assets for faster loading
- Located at `public/sw.js` (auto-generated on build)

### 3. App Icons
- Money icon design with green color scheme
- SVG icon: `public/icons/icon.svg`
- PNG icons: 192x192 and 512x512 sizes
- Located in `public/icons/`

### 4. PWA Features Enabled
- Installable on mobile devices (iOS and Android)
- Installable on desktop (Chrome, Edge, etc.)
- Offline support
- Fast loading with service worker caching
- Standalone app experience (no browser UI)
- Theme color and splash screen

## How to Test

### On Desktop
1. Run `npm run build` and `npm start`
2. Open Chrome/Edge browser
3. Visit your app URL
4. Look for the install icon in the address bar
5. Click to install as desktop app

### On Mobile
1. Deploy your app to a production URL (must use HTTPS)
2. Visit the URL on your mobile device
3. iOS: Tap Share → Add to Home Screen
4. Android: Tap the "Install" banner or menu → Install app

## Improving the Icons

The current icons are basic placeholders. To create professional icons:

1. **Option 1: Use the SVG**
   - Open `public/icons/icon.svg` in a design tool (Figma, Adobe Illustrator)
   - Customize the money icon design
   - Export as PNG in required sizes

2. **Option 2: Online Tools**
   - Use [PWA Asset Generator](https://www.pwabuilder.com/imageGenerator)
   - Upload a square 512x512 PNG
   - Download all generated icon sizes

3. **Option 3: Design Custom**
   - Create a 512x512 PNG icon in your design tool
   - Use tools like [RealFaviconGenerator](https://realfavicongenerator.net/)
   - Replace files in `public/icons/`

## Build Process

The PWA files are generated automatically when you run:
```bash
npm run build
```

In development mode (`npm run dev`), the PWA is disabled to avoid caching issues.

## Deployment Notes

- Ensure your production URL uses HTTPS (required for PWAs)
- The service worker will only work on HTTPS or localhost
- Update `start_url` in `manifest.json` if deploying to a subdirectory
- Test on multiple devices and browsers

## Customization

### Change Theme Color
Edit `app/layout.tsx`:
```typescript
export const viewport: Viewport = {
    themeColor: '#10B981', // Change this color
    // ...
};
```

Also update `public/manifest.json`:
```json
{
  "theme_color": "#10B981", // Change this color
  "background_color": "#ffffff" // And this if needed
}
```

### Modify App Name or Description
Edit `public/manifest.json`:
```json
{
  "name": "ArtePay",
  "short_name": "ArtePay",
  "description": "Your custom description"
}
```

### Add More Shortcuts
Edit the `shortcuts` array in `public/manifest.json` to add quick actions.

## Troubleshooting

### PWA not installing?
- Make sure you're using HTTPS (or localhost)
- Check browser console for errors
- Verify `manifest.json` is accessible at `/manifest.json`
- Clear browser cache and service worker

### Icons not showing?
- Ensure icon files exist in `public/icons/`
- Check file paths in `manifest.json`
- Verify icon sizes are correct (192x192, 512x512)

### Service Worker not updating?
- Run `npm run build` again
- Clear application cache in browser DevTools
- Unregister old service workers in DevTools → Application → Service Workers
